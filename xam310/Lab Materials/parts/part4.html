Data Binding in Xamarin.Forms

Duration

15 minutes

Goals

This homework assignment will continue modifying our Fun Flag Facts application and move the testable code out of the XAML code behind - and provide the logic for the screen as a fully segregated class. You should always do this as a best practice because it will allow you to put a different UI onto the app (for example a native iOS view) but keep the tested logic independent.

Assets

This exercise includes a completed project in the Part 04 Resources folder if you'd like to check your work. If you need a starting point, you can use the Completed folder in the prior exercise.

Assignment

See if you can pull the basic flag state out of the code behind and put it into a shared class in the FlagData project. Think about how this class could use INotifyPropertyChanged to update the BindingContext. You will need to leave the event handlers in place for the buttons - but the actual logic can be shifted into a shared class, sometimes referred to as a "ViewModel". Check out the completed lab for a full implementation - it has a new class in the FlagData project named FunFlactsViewModel.
