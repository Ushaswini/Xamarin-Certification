Before you begin

This is a Xamarin.Forms class which includes projects for iOS, Android, and Windows. When working on the exercises, you can select the easiest supported platform to run your application on. Refer to the Setup Requirements on the Xamarin University website if you need help with your development environment.

Overview

This session introduces the data binding infrastructure that is built into Xamarin.Forms. You will learn how to take advantage of this feature in your Xamarin.Forms applications to reduce the amount of code necessary to present data in your screens.

Objectives

  1. Use data binding to decouple code-behind from UI
  2. Use value converters to bind incompatible types

Lecture

XAM310 - Data Binding in Xamarin.Forms.

Exercises

  1. Using Data Binding in a Xamarin.Forms Application
    Exercise
    This group exercise will introduce creating and using bindings in code-behind and XAML.
  2. Keeping the UI and data in sync using Bindings
    Exercise
    This exercise will implement two-way binding for our UI; this will ensure UI data changes are sent to the code-behind. It will then implement INotifyPropertyChanged to ensure that the UI is updated when the underlying data is changed.
  3. Using Value Converters
    Exercise
    There isn't always a 1:1 mapping between our available data and the user interface that presents that data. Value Converters are the bridge which spans this difference. In this exercise, we will use a Value Converter to data bind images to our resources.
  4. Homework: abstracting out the shared code
    Homework
    In this homework assignment, you will pull all the sharable code out of the code behind now that it no longer depends on the UI controls and place it into a separate class which can be unit tested.
